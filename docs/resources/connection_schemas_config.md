# Resource: fivetran_connection_schemas_config

~> **Preview** This resource is in preview. Its behavior and interface may change in future releases without prior notice.

Manages the schema change handling policy and schema-level enable/disable settings for a Fivetran connection.

The list you choose defines the complete desired state for all schemas:

- **`disabled_schemas`** — listed schemas are **disabled**, all other schemas are **enabled**
- **`enabled_schemas`** — listed schemas are **enabled**, all other schemas are **disabled**

Either attribute can be used with any `schema_change_handling` policy. If a new schema appears upstream, the resource will automatically apply the correct state based on which list you configured.

## Example Usage

### Disable specific schemas (all others enabled)

```hcl
resource "fivetran_connection_schemas_config" "config" {
  connection_id          = fivetran_connector.my_connector.id
  schema_change_handling = "ALLOW_ALL"
  disabled_schemas       = ["analytics", "staging"]
}
```

### Enable specific schemas (all others disabled)

```hcl
resource "fivetran_connection_schemas_config" "config" {
  connection_id          = fivetran_connector.my_connector.id
  schema_change_handling = "BLOCK_ALL"
  enabled_schemas        = ["public", "reporting"]
}
```

-> **NOTE:** The `disabled_schemas` and `enabled_schemas` attributes conflict with each other — use one or the other.

## Schema

### Required

- `connection_id` (String) The unique identifier for the connection within the Fivetran system.
- `schema_change_handling` (String) The value specifying how new source schema changes are handled. One of: ALLOW_ALL, ALLOW_COLUMNS, BLOCK_ALL.

### Optional

- `disabled_schemas` (List of String) Set of schema names to disable. Use when schema_change_handling is ALLOW_ALL.
- `enabled_schemas` (List of String) Set of schema names to enable. Use when schema_change_handling is BLOCK_ALL or ALLOW_COLUMNS.

### Read-Only

- `id` (String) The unique resource identifier (equals to connection_id).
